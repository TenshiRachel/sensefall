# Vercel Cloud Hosting Setup (SmartFall)

This setup hosts a remote dashboard on Vercel while fall detection stays on Raspberry Pi.

## Architecture

- Raspberry Pi: local detection + local email + optional cloud sync
- Vercel: hosted API + hosted remote dashboard UI
- Supabase: persistent event storage

## 1. Create Supabase Table

Run this SQL in Supabase SQL editor:

```sql
create table if not exists smartfall_events (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  event_time text not null,
  event_type text not null,
  confidence double precision,
  device_id text,
  source text,
  metadata jsonb default '{}'::jsonb
);
```

## 2. Deploy to Vercel

1. Push this branch to GitHub
2. Import repo into Vercel
3. Deploy with root settings as-is

## 3. Set Vercel Environment Variables

Add in Vercel Project Settings:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `CLOUD_API_KEY` (any strong random string)

## 4. Test Cloud API

After deploy:

- `GET https://<your-vercel-domain>/api/health`
- `GET https://<your-vercel-domain>/api/events`

## 5. Connect Raspberry Pi to Cloud

On Pi `.env`, add:

```env
CLOUD_SYNC_URL=https://<your-vercel-domain>
CLOUD_SYNC_API_KEY=<same CLOUD_API_KEY from vercel>
CLOUD_DEVICE_ID=home_pi_01
CLOUD_SYNC_TIMEOUT_SEC=5
```

Then restart Pi dashboard:

```bash
set -a; source .env; set +a
python3 dashboard.py
```

## 6. Parent Remote Access

Share your Vercel URL with parent:

- `https://<your-vercel-domain>/`

They can view event logs outside home network.

## Notes

- Vercel does not run camera or edge AI detection.
- Keep detection local on Raspberry Pi.
- Cloud stores only event data, not raw video.
